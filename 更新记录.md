## 2026-05-10 版本更新

本次改动集中在三个方向：
1. 数据源扩展（新增东方财富资金流向）
2. 双循环监控架构
3. AI 智能点评（DeepSeek）

---

### 改动清单

#### fetcher.py — 新增 3 个函数

- `_request_with_retry()` — 带重试的 HTTP 请求（原有函数只有单次请求）
- `fetch_sector_moneyflow()` — 东方财富板块资金流向（主力净流入/占比/大中小单，单位亿元）
- `fetch_stock_moneyflow_top(n)` — 个股资金流向 Top N
- `fetch_top_stocks_by_amount(n)` — 成交额排名前N只股票（用于快循环热点池）

**数据字段说明：**

| 字段 | 说明 | 来源API |
|------|------|---------|
| main_net_inflow | 主力净流入(亿元) | 东方财富 f62 |
| main_net_ratio | 主力净占比(%) | 东方财富 f184 |
| big_net_inflow | 大单净流入(亿元) | 东方财富 f66 |
| mid_net_inflow | 中单净流入(亿元) | 东方财富 f72 |
| small_net_inflow | 小单净流入(亿元) | 东方财富 f78 |

主力 = 超大单 + 大单。正值为净流入（看多），负值为净流出（看空）。

---

#### main.py — 双循环架构重构

旧版：单循环，固定间隔扫全部5000只

新版双循环：
- **快循环（60秒）**：扫成交额Top300热点池 → 涨跌超5%自动弹macOS通知
- **慢循环（300秒）**：全量5000只 + 资金流向 + 调DeepSeek生成AI点评

CLI 参数更新：
- `--fast N` — 快循环间隔（默认60秒）
- `--slow N` — 慢循环间隔（默认300秒）
- `--once` — 单次扫描 + AI点评
- 默认启动进入双循环监控模式

---

#### ai_advisor.py — 新增文件

DeepSeek 原生 API 调用模块（OpenAI 兼容协议）。

- API Key 从环境变量读取：优先 `DEEPSEEK_API_KEY`，其次 `ANTHROPIC_AUTH_TOKEN`
- 调用模型：`deepseek-chat`（实际运行是 deepseek-v4-flash）
- 调用时机：每轮慢循环自动触发，或用户在仪表盘点击"生成点评"
- 生成内容：市场整体判断 + 资金动向解读 + 关注方向建议（200字内）

---

#### webui.py — 前端适配

新增区域：
- **资金流向看板**：板块主力净流入/流出 TOP 8 表格
- **AI 点评卡片**：点"生成点评"按钮调 DeepSeek 分析
- **扫描进度条**：实时显示扫描状态（热点池/全量/完成）
- **前端刷新**：从30秒缩短到10秒

---

#### launch.sh — 启动脚本更新

新增 `--dual` 模式：同时启动 Web 仪表盘 + 双循环监控。

---

### 已知问题

- 全量扫描5000只腾讯行情耗时约30-60秒（网络I/O瓶颈）
- 热点池扫描300只耗时约2-5秒
- 非交易日无数据（依赖A股交易时间判断）
- API key 通过环境变量传入，需在 settings.json 中配置

### 如何运行

```bash
# Web 仪表盘（含资金流向 + AI点评 + 进度条）
cd ~/MarketPulse && python3 webui.py --port 9999

# 单次扫描 + AI 点评
cd ~/MarketPulse && python3 main.py --once

# 双循环监控（终端模式）
cd ~/MarketPulse && python3 main.py

# 启动脚本
cd ~/MarketPulse && ./launch.sh --web      # 仪表盘
cd ~/MarketPulse && ./launch.sh --dual     # 仪表盘 + 双循环
```
